# API Reference

The `@flowindex/agent-wallet` MCP server exposes the following MCP tools.

## Wallet Tools

### `wallet_status`

Returns the current wallet configuration and status.

**Parameters:** None

**Returns:**
```json
{
  "signer_type": "local",
  "flow_address": "1234567890abcdef",
  "evm_address": "0x...",
  "key_index": 0,
  "sig_algo": "ECDSA_secp256k1",
  "hash_algo": "SHA2_256",
  "network": "mainnet",
  "approval_required": true,
  "headless": true
}
```

### `wallet_login`

Initiates an interactive login flow for the cloud wallet. Returns a login URL the user must visit.

**Parameters:** None

**Returns:**
```json
{
  "status": "pending",
  "login_url": "https://flowindex.io/wallet/login?session=...",
  "session_id": "abc-123",
  "expires_in": 300
}
```

### `wallet_login_status`

Checks the status of an interactive login session. If authenticated, activates the cloud signer.

**Parameters:**

| Name | Type | Description |
|------|------|-------------|
| `session_id` | string | The session ID returned by `wallet_login` |

**Returns (authenticated):**
```json
{
  "status": "authenticated",
  "flow_address": "1234567890abcdef",
  "key_index": 0
}
```

### `diagnose_signing`

Runs a comprehensive diagnostic on the signing setup: verifies the configured private key matches the on-chain account key, checks sigAlgo/hashAlgo, weight, revocation status, and performs a local sign+verify test. Use this to debug "invalid signature" errors.

**Parameters:** None

**Returns:**
```json
{
  "summary": "All checks passed — signing setup looks correct.",
  "checks": [
    { "check": "signer_type", "status": "pass", "detail": "Type: local-key, Address: 0x..., KeyIndex: 0, ..." },
    { "check": "local_public_key", "status": "pass", "detail": "04abc...def (128 hex chars)" },
    { "check": "key_weight", "status": "pass", "detail": "Weight: 1000 (need >= 1000 for full signing authority)" },
    { "check": "sig_algo_match", "status": "pass", "detail": "Both: ECDSA_secp256k1" },
    { "check": "hash_algo_match", "status": "pass", "detail": "Both: SHA2_256" },
    { "check": "public_key_match", "status": "pass", "detail": "Keys match (04abc...)" },
    { "check": "local_sign_verify", "status": "pass", "detail": "Signature produced: ... (128 hex chars, correct format)" }
  ]
}
```

Checks performed:
- **signer_type**: Basic signer info (type, address, key index, algorithms)
- **local_public_key**: Public key derivation from the private key (local signers only)
- **onchain_key_fetch**: Fetches account keys from the Flow Access API
- **key_revoked**: Whether the key index is revoked on-chain
- **key_weight**: Key weight must be >= 1000 for full signing authority
- **sig_algo_match**: Configured signing algorithm matches on-chain key
- **hash_algo_match**: Configured hashing algorithm matches on-chain key
- **public_key_match**: Local public key matches on-chain public key
- **local_sign_verify**: Local sign+verify round-trip test

## Template Tools

### `list_templates`

Lists available Cadence transaction and script templates.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `category` | string | No | Filter by category such as `token`, `collection`, `evm`, or `bridge` |

**Returns:** Array of template summaries with `name`, `category`, `type`, `description`, and `arg_count`.

### `get_template`

Retrieves the full Cadence source code and argument schema for a named template.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `name` | string | Yes | Template name such as `transfer_tokens_v3` |

**Returns:** Template object with `name`, `category`, `type`, `description`, `cadence`, and `args`.

### `execute_script`

Executes a read-only Cadence script on the Flow network using a named template.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `template_name` | string | Yes | Name of a script template |
| `args` | object | No | Named arguments matching the template arg schema |

### `execute_cadence_script`

Executes a read-only Cadence script from raw source code. This is the freeform alternative to `execute_script`.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `cadence` | string | Yes | Raw Cadence script source code |
| `arguments` | array | No | Ordered typed arguments in `{ type, value }` form |

**Returns:**
```json
{
  "result": {
    "ok": true
  }
}
```

### `simulate_cadence_transaction`

Runs a read-only preflight simulation for a raw Cadence transaction using the configured wallet address as both authorizer and payer.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `cadence` | string | Yes | Raw Cadence transaction source code |
| `arguments` | array | No | Ordered typed arguments in `{ type, value }` form |
| `scheduled` | object | No | Optional scheduled replay controls |

**`scheduled` fields:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `advance_seconds` | number | No | Wait time before replaying scheduled work, max `5` |
| `advance_blocks` | number | No | Extra blocks to advance, max `20` |

**Returns:**
```json
{
  "simulation": {
    "success": true,
    "summary": "FLOW transfer"
  }
}
```

### `simulate_template`

Runs a read-only preflight simulation for a transaction template on the FlowIndex simulator. This does not sign or submit anything.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `template_name` | string | Yes | Name of the transaction template |
| `args` | object | Yes | Named arguments matching the template arg schema |
| `scheduled` | object | No | Optional scheduled replay controls |

**`scheduled` fields:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `advance_seconds` | number | No | Wait time before replaying scheduled work, max `5` |
| `advance_blocks` | number | No | Extra blocks to advance, max `20` |

**Returns:**
```json
{
  "success": true,
  "summary": "FLOW transfer",
  "transfers": [],
  "nftTransfers": [],
  "balanceChanges": [],
  "scheduledResults": []
}
```

### `execute_cadence_transaction`

Submits a raw Cadence transaction with typed arguments. On mainnet, the server first attempts a non-blocking preflight simulation. Headless raw Cadence signing is approval-gated by default unless `ALLOW_RAW_CADENCE_SIGNING=true`.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `cadence` | string | Yes | Raw Cadence transaction source code |
| `arguments` | array | No | Ordered typed arguments in `{ type, value }` form |
| `scheduled` | object | No | Optional scheduled replay controls for the preflight simulation |

**Returns (immediate execution):**
```json
{
  "status": "sealed",
  "tx_id": "abc123...",
  "block_height": 12345678,
  "events": [],
  "preflight_simulation": {
    "success": true,
    "summary": "FLOW transfer",
    "error": null
  }
}
```

**Returns (approval required):**
```json
{
  "status": "pending_approval",
  "tx_id": "uuid",
  "summary": "transaction(amount: UFix64, to: Address) [arg_count=2]",
  "message": "Raw Cadence headless signing is disabled by default. Transaction queued for approval."
}
```

### `execute_template`

Executes a Cadence transaction template. On mainnet, the server first attempts a non-blocking preflight simulation when simulator calls are enabled. If approval is required and the signer is headless, the transaction is queued for manual approval.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `template_name` | string | Yes | Name of the transaction template |
| `args` | object | Yes | Named arguments matching the template arg schema |

**Returns (immediate execution):**
```json
{
  "status": "sealed",
  "tx_id": "abc123...",
  "block_height": 12345678,
  "events": [{ "type": "A.xxx.FlowToken.TokensWithdrawn", "data": {} }],
  "preflight_simulation": {
    "success": true,
    "summary": "FLOW transfer",
    "error": null
  }
}
```

**Returns (approval required):**
```json
{
  "status": "pending_approval",
  "tx_id": "uuid",
  "summary": "transfer_tokens_v3(recipient=0x..., amount=1.0)",
  "preflight_simulation": {
    "success": true,
    "summary": "FLOW transfer",
    "error": null
  }
}
```

If simulation is unavailable, the response may contain `preflight_simulation_skipped_reason` or `preflight_simulation_error` instead. Those cases do not block the real transaction flow.

## Approval Tools

### `confirm_transaction`

Approves and executes a pending transaction.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `tx_id` | string | Yes | The pending transaction ID |

**Returns:** Same as `execute_template` immediate execution result, including any stored `preflight_simulation`, `preflight_simulation_skipped_reason`, or `preflight_simulation_error`.

### `cancel_transaction`

Cancels and discards a pending transaction.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `tx_id` | string | Yes | The pending transaction ID |

### `list_pending`

Lists all transactions currently queued and awaiting approval.

**Parameters:** None

**Returns:**
```json
{
  "count": 1,
  "transactions": [
    {
      "tx_id": "uuid",
      "kind": "template",
      "summary": "transfer_tokens_v3(...)",
      "created_at": 1700000000000,
      "preflight_simulation": {
        "success": true,
        "summary": "FLOW transfer",
        "error": null
      }
    }
  ]
}
```

## Flow Query Tools

### `get_account`

Returns detailed information about a Flow account including keys, contracts, and storage usage.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `address` | string | Yes | Flow address, with or without `0x` prefix |

### `get_flow_balance`

Returns the native FLOW token balance for a Flow account.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `address` | string | Yes | Flow address |

### `get_ft_balance`

Returns all fungible token vault balances for a Flow account.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `address` | string | Yes | Flow address |

### `get_nft_collection`

Returns NFT collections and items held by a Flow account.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `address` | string | Yes | Flow address |

### `get_transaction`

Returns full details for a Flow transaction by its ID, including events, status, and error messages.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `tx_id` | string | Yes | Flow transaction ID |

## EVM Tools

### `evm_wallet_address`

Returns the current EVM EOA address derived from the configured signer.

**Parameters:** None

### `evm_get_balance`

Returns the native FLOW balance of an EVM address on Flow EVM.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `address` | string | Yes | EVM address |

### `evm_get_token_balance`

Returns the ERC-20 token balance for an address, including symbol and decimals.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `token_address` | string | Yes | ERC-20 token contract address |
| `owner` | string | Yes | Address to check balance for |

### `evm_transfer`

Sends native FLOW on Flow EVM from the agent wallet to a recipient address.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `to` | string | Yes | Recipient EVM address |
| `amount` | string | Yes | Amount in FLOW, for example `"1.5"` |

### `evm_transfer_erc20`

Transfers an ERC-20 token on Flow EVM from the agent wallet to a recipient.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `token_address` | string | Yes | ERC-20 token contract address |
| `to` | string | Yes | Recipient EVM address |
| `amount` | string | Yes | Amount in human-readable units, for example `"100.0"` |
| `decimals` | number | No | Token decimals, default `18` |

### `evm_read_contract`

Calls a read-only function on an EVM contract.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `contract_address` | string | Yes | Contract address |
| `abi` | string | Yes | ABI as a JSON array string |
| `function_name` | string | Yes | Function name to call |
| `args` | string | No | Arguments as a JSON array string |

### `evm_write_contract`

Calls a state-changing function on an EVM contract.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `contract_address` | string | Yes | Contract address |
| `abi` | string | Yes | ABI as a JSON array string |
| `function_name` | string | Yes | Function name to call |
| `args` | string | No | Arguments as a JSON array string |
| `value` | string | No | Native FLOW to send, for example `"0.5"` |

### `evm_get_transaction`

Returns details of an EVM transaction by its hash.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `tx_hash` | string | Yes | Transaction hash |
